# Sanmoo Blog 后端冻结能力清单

## 1. 冻结策略说明

### 1.1 冻结定义

冻结能力指当前系统中存在但不再主动维护和扩展的功能模块。冻结策略遵循以下原则：

- **接口保留**：相关API仍保持可用，避免前端直接崩溃
- **数据冻结**：不再主动触发相关后台任务和数据更新
- **权限弱化**：后台管理端相关权限标记为"低频/冻结"
- **文档标记**：接口文档明确标注为"已冻结，不建议继续使用"
- **渐进移除**：待前端完成适配后，可在后续版本中考虑完全移除

### 1.2 冻结原因分类

| 原因 | 说明 | 示例 |
|------|------|------|
| 平台化扩张 | 功能偏向企业级平台运营，不符合个人站点定位 | 用户管理、角色权限管理 |
| 重运营画像 | 功能涉及复杂用户画像和标签运营，合规风险高 | 用户画像、雷达图、标签生成 |
| 低频运维 | 功能属于运维工具，个人站点使用频率极低 | 缓存管理、备份管理、日志清理 |
| 合规风险 | 功能可能超出个人备案业务边界 | 评论社区、投稿系统、交易功能 |

## 4. 冻结能力前端适配建议

### 4.1 后台菜单隐藏

以下菜单入口应在前端隐藏：

- 用户管理（完整隐藏）
- 角色管理（完整隐藏）
- 权限管理（完整隐藏）
- 小程序用户管理（完整隐藏）
- 缓存管理（完整隐藏）
- 备份管理（完整隐藏）
- 数据维护（完整隐藏）

### 4.2 后台页面弱化

以下页面应在前端弱化显示或设置访问限制：

- 仪表盘：隐藏访客记录管理、错误日志管理、小程序用户增长
- 设置中心：隐藏配置导入导出功能

### 4.3 小程序端弱化

以下功能应在小程序端弱化或隐藏：

- 用户画像展示
- 复杂推荐入口
- 用户标签管理
