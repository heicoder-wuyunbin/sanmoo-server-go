# Sanmoo Blog 后端业务边界文档

## 1. 项目定位

> **个人技术内容发布与知识整理平台**

以"个人原创内容发布、专题沉淀、文章检索、轻量化小程序阅读分发"为主，不做开放投稿、不做评论社区、不做商业交易、不做会员付费、不做用户社交。

---

## 2. 个人备案约束

### 2.1 必须遵守的边界

| 约束项 | 说明 | 状态 |
|--------|------|------|
| 站点属性 | 明确为个人原创内容站 | ✅ 已确认 |
| 公众注册 | 不开放公众注册成为站内运营主体 | ✅ 已确认 |
| 用户投稿 | 不开放用户投稿与公开发布 | ✅ 已确认 |
| 评论社区 | 不开放评论社区 | ✅ 已确认 |
| 交易功能 | 不做交易、充值、会员付费 | ✅ 已确认 |
| 用户数据 | 小程序用户数据仅用于阅读体验与基础功能 | ✅ 已确认 |
| 隐私政策 | 提供清晰的隐私政策与用户删除路径 | ✅ 已确认 |
| 备案展示 | 展示备案信息、联系方式、隐私政策入口 | ⏳ 待实现 |

### 2.2 禁止新增或强化的业务

- 评论社区
- 用户发帖或投稿系统
- 商业会员体系
- 订单、支付、充值
- 社交互动链路
- 用户等级成长系统
- 重运营型推荐后台
- 面向大量用户的开放式社区运营

---

## 3. 业务域划分与保留策略

### 3.1 内容核心层（保留并强化）

负责长期价值沉淀，只围绕内容资产建设：

| 能力项 | 说明 | 策略 |
|--------|------|------|
| 文章管理 | 文章CRUD、定时发布、SEO slug | 保留并优化 |
| 分类管理 | 文章分类组织 | 保留 |
| 标签管理 | 文章标签组织 | 保留 |
| 专题管理 | 专题编排、文章关联、排序 | **强化**（一级运营能力） |
| 归档浏览 | 按时间归档文章 | 保留 |
| SEO优化 | JSON-LD、面包屑、sitemap | 保留 |
| RSS订阅 | 内容聚合输出 | 保留 |
| 搜索功能 | MeiliSearch全文检索 | 保留 |
| 相关推荐 | 基于标签/分类的轻量推荐 | 保留（降为规则推荐） |

### 3.2 渠道分发层（保留并收敛）

只承担内容消费，不承载复杂平台化运营：

#### Web 渠道

| 能力项 | 说明 | 策略 |
|--------|------|------|
| 文章浏览 | 列表、详情、分类、标签 | 保留 |
| 专题阅读 | 专题列表、专题详情 | 保留 |
| 搜索 | 全文搜索、热门搜索 | 保留 |
| 归档 | 时间轴归档 | 保留 |
| 点赞 | 文章点赞 | 保留 |
| 友情链接 | 白名单管理 | 保留 |

#### 小程序渠道

| 能力项 | 说明 | 策略 |
|--------|------|------|
| 登录态 | 微信小程序授权登录 | 保留 |
| 阅读 | 文章浏览、专题阅读 | 保留 |
| 收藏 | 文章收藏管理 | 保留 |
| 历史 | 浏览历史记录 | 保留 |
| 订阅 | 订阅状态管理 | 保留 |
| 推荐 | 轻量规则推荐 | 保留（弱化） |
| 用户删除 | 数据删除路径 | 保留 |

### 3.3 轻运营层（保留并精简）

只保留个人站长真正会长期使用的能力：

| 能力项 | 说明 | 策略 |
|--------|------|------|
| 内容管理 | 文章/分类/标签/专题/链接管理 | **核心保留** |
| 文件管理 | 图片、附件上传管理 | 保留 |
| 搜索配置 | MeiliSearch配置、热搜索配置 | 保留 |
| 定时发布 | 文章定时发布 | 保留 |
| 数据看板 | 内容表现统计 | 保留（聚焦内容） |
| 友情链接 | 白名单管理 | 保留 |

### 3.4 合规与基础设施层（保留并补齐）

对用户可感知但不过度扩张：

| 能力项 | 说明 | 策略 |
|--------|------|------|
| 隐私政策 | 隐私政策内容管理 | **补齐** |
| 数据删除 | 用户数据删除接口 | 保留 |
| 备案展示 | 备案信息展示字段 | **补齐** |
| 日志策略 | 日志保留与清理 | **补齐** |
| 搜索 | MeiliSearch基础能力 | 保留 |
| 缓存 | Redis缓存基础能力 | 保留 |
| 存储 | 本地/OSS存储配置 | 保留 |
| 邮件 | SMTP邮件配置 | 保留 |

## 4. 小程序用户域收敛策略

### 4.1 保留能力

| 能力项 | 数据表 | 说明 |
|--------|--------|------|
| 用户基础信息 | `t_mp_user` | 登录态、昵称、头像 |
| 收藏 | `t_mp_user_favorite` | 文章收藏管理 |
| 浏览历史 | `t_mp_browse_history` | 阅读历史记录 |
| 订阅状态 | `t_mp_user_subscribe` | 订阅开关 |
| 行为日志 | `t_mp_user_behavior` | 基础行为记录（用于内容优化） |
| 用户删除 | `DELETE /mp/user` | 数据删除路径 |

### 4.2 弱化能力（冻结扩张，暂不删除）

| 能力项 | 数据表 | 说明 |
|--------|--------|------|
| 用户兴趣画像 | `t_mp_user_interest` | 兴趣分、维度得分 |
| 用户标签 | `t_mp_user_tag` | 用户标签生成与管理 |
| 用户六边形画像 | `t_mp_user_profile` | 多维度画像得分 |
| 推荐曝光记录 | `t_mp_reco_exposure` | 推荐曝光追踪 |

### 4.3 弱化策略说明

- **接口保留**：相关接口仍保持可用，避免前端直接崩溃
- **数据冻结**：不再主动触发画像生成、标签更新等后台任务
- **权限弱化**：后台管理端相关权限标记为"低频/冻结"
- **文档标记**：接口文档明确标注为"已冻结，不建议继续使用"

## 6. 数据治理原则

### 6.1 数据保留策略

| 数据类型 | 保留期限 | 说明 |
|----------|----------|------|
| 文章内容 | 长期 | 核心资产 |
| 用户行为日志 | 30天 | 用于内容优化 |
| 访问日志 | 30天 | 用于运维排查 |
| 错误日志 | 90天 | 用于问题追踪 |
| 小程序用户画像数据 | 按用户删除请求清理 | 不主动保留 |

### 6.2 数据用途约束

- 行为数据仅用于内容优化，不作为用户运营资产
- 用户画像数据仅用于轻量推荐，不对外暴露
- 不收集或存储用户社交关系数据
- 不收集或存储用户支付、交易相关数据

## 7. 配置域重构方向

配置域重构为四类，详见 `config-domain-mapping.md`：

1. **站点品牌配置**：博客名、作者、介绍、头像、站点地址、SEO基础项
2. **合规配置**：隐私政策、数据保留说明、账号注销说明、备案信息展示
3. **渠道配置**：Web渠道配置、小程序渠道配置、社交链接展示开关
4. **基础设施配置**：搜索、存储、邮件、缓存等技术配置

---

## 8. 权限模型收敛方向

用户角色收敛为两级：

| 角色 | 说明 |
|------|------|
| 站长 | 拥有全部权限，负责内容生产与站点管理 |
| 编辑 | 拥有内容编辑权限，负责文章撰写与编排 |

不再继续扩展复杂 RBAC 维度。
